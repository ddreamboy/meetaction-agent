## MeetAction Agent (PoC)

### Что за задача, для кого и какая боль сейчас
* **Задача:** Автоматизация менеджмента итогов встреч
* **Для кого:** Проджект-менеджеры, тимлиды, аналитики и любые сотрудники, регулярно участвующие в рабочих созвонах
* **Боль:** Сейчас после созвонов нужно вручную прослушивать записи/вспоминать детали, извлекать договоренности и руками заводить задачи в трекерах. Это приводит к потере важного контекста, забытым обещаниям и немалой трате времени (до 4 часов в неделю на рутину)

### Что именно сделает PoC на демо
Пользователь загружает аудио/видео рабочей встречи. Система транскрибирует запись и извлекает граф знаний (кто, что, кому обещал). Затем Агент анализирует граф, самостоятельно формирует список тасок, предлагает их пользователю и, получив "ОК" (human-in-the-loop), создает карточки в тестовом таск-трекере (например, Trello/Notion) через API. Также агент может ответить на вопросы по базе прошлых встреч (RAG).

### Что НЕ делает PoC (out-of-scope)
- Не интегрируется с Zoom/Meet/Teams в реальном времени (работает только с записями)
- Не поддерживает распознавание эмоций и интонаций участников
- Не назначает звонки в календарь (только таск-трекинг)

### Запуск

**1. Создайте `.env`:**
```env
LLM_BASE_URL=https://openrouter.ai/api/v1
LLM_API_KEY=sk-...
LLM_MODEL=google/gemini-2.5-flash-lite

TODOIST_API_TOKEN=...
TODOIST_PROJECT_ID=...  # опционально

TRANSCRIBE_BACKEND=deepgram
DEEPGRAM_API_KEY=...
```

**2. Запустите:**

```bash
# Lite (по умолчанию) - без whisperx, транскрибация через Deepgram API
docker compose up --build

# Full - с локальным Whisper (нужен GPU или мощный CPU)
docker compose -f docker-compose.yml -f docker-compose.full.yml up --build
```

Сервисы:
- **API** - `http://localhost:7007`
- **UI** - `http://localhost:7860`
- Redis и Qdrant поднимаются автоматически

### Как пользоваться

**Вкладка "Обработка встречи":**
1. Загрузите аудио/видео файл **или** вставьте готовую транскрипцию в текстовое поле.
2. Дождитесь обработки - агент извлечёт задачи и покажет предложенный список.
3. Задайте имена спикеров в таблице и нажмите "Применить имена".
4. При необходимости скорректируйте задачи в чате - нажмите "Применить правки".
5. Нажмите "Подтвердить и создать задачи в Todoist".

**Вкладка "Вопросы по встречам":**
- Задавайте вопросы на естественном языке по всем проиндексированным встречам: "Кто обещал сдать отчёт?", "Какие задачи были у Артёма?"
- Фильтруйте по типу встречи (рабочие / консультации).
