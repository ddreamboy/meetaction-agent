**Предложено (4 шт.):**
  - **RAG системы**
    - RAG (Retrieval-Augmented Generation) — это подход, который сначала ищет релевантные документы в векторной базе данных, а затем передает их в контекст большой языковой модели (LLM) для генерации ответа.
  - **Применение RAG**
    - RAG позволяет LLM отвечать на вопросы, используя специфические внутренние данные компании, без необходимости дообучения самой модели.
  - **Сложности RAG**
    - Ключевыми сложностями при работе с RAG являются качество разбиения документов на чанки (chunking) и подбор порога схожести для поиска релевантных документов.
  - **Рекомендации для старта с RAG**
    - Для начала работы с RAG рекомендуется использовать векторные базы данных Qdrant или Chroma, embedding-модели от OpenAI или Gemini, а также начать с косинусного сходства (cosine similarity) с порогом 0.7-0.8.

    - Structured JSON logs and correlation IDs are recommended starting points for improving observability.
  - **Correlation ID**
    - A correlation ID is a unique identifier that tracks a request across multiple services.
    - It allows for the reconstruction of a request's complete journey by aggregating information from various logs.
  - **Metrics**
    - Key metrics for monitoring include RED metrics (Rate, Errors, Duration).
