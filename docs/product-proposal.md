## Обоснование идеи
Автоматизация процесса извлечения договоренностей из рабочих встреч и заведения соответствующих задач в таск-трекере

**Проблематика и боль**

В современных распределенных командах сотрудники проводят значительную часть рабочего времени на онлайн-встречах. После каждого созвона от менеджера или участника требуется время на составление "итогов" встречи лично для себя и для всех (в случае менеджера) и ручное выделение ключевых договоренностей и перенос их в системы управления проектами 
* Боли:
    * **трата времени** - на ручную обработку результатов встреч может уходить до 4 часов в неделю у одного специалиста
    * **потеря контекста и человеческий фактор** - участники часто забывают, кто именно и что пообещал сделать, сроки размываются, часть договоренностей теряется из-за усталости и потери фокуса внимания в процессе встречи

**Предлагаемое решение**

Система, которая берет на себя эту рутину, транскрибирует аудио/видео записи, анализирует диалог и строит структурированный граф знаний (субъект -> действие -> объект -> дедлайн). На основе этого графа Агент самостоятельно формирует список задач. Это позволит разгрузить процесс менеджмента, бизнесу больше не нужно тратить часы на постобработку встреч, а риск забытых обещаний сводится к нулю. В долгосрочной перспективе такая фиксация рабочих встреч позволит проводить аналитику по их качеству и, в случае чего, улучшать их организацию и проведение.

## Цель и метрики
* Продуктовая:
    * % задач, которые пользователь аппрувит без редактирования (цель > 80%)
* Техническая:
    * успешное сведение графа сущностей без дубликатов на текстах > 20к токенов
    * удержание p95 latency агента в пределах 30 секунд

## Потенциальные сценарии использования
**Happy path:**
1. Пользователь загружает аудио/видео-запись встречи в интерфейс
2. Система транскрибирует аудио и извлекает граф договоренностей с помощью LLM (если на шаге 1 было загружено видео, то предварительно из него извлекается аудио) 
3. Агент генерирует список задач (название, описание, assignee, дедлайн) и показывает пользователю
4. Пользователь нажимает "Подтвердить" -> задачи создаются в таск-трекере

**Edge-кейсы:**
* *Неразборчивая речь или нет четких договоренностей* - агент сообщает пользователю, что конкретные задачи извлечь не удалось, и предлагает краткое саммари созвона
* *Отказ пользователя* - пользователь нажимает "Отклонить" и пишет "Сделай задачи более детализированными", агент переделывает список задач с учетом фидбэка
* *Превышение контекста (очень долгий созвон)* - cистема разбивает транскрипт на чанки (map-reduce) для извлечения графа

## Ограничения
* **Технические**
    * аудиозаписи ограничены 90 минутами для PoC (чтобы не упираться в лимиты токенов)
    * p95 latency агента (без учета транскрибации) не более 30 секунд
* **Операционные**
    * бюджет на API LLM не более $15-20 на время разработки и тестирования PoC

## Архитектурный набросок
* **Frontend** - простой UI для загрузки файлов и взаимодействия с агентом
* **Модуль транскрибации** на базе Whisper
* **Агент** на LangChain / LangGrapgh, включает логику графового извлечения и формирования задач
* **State и memory** в Redis
* **Интеграции tools** через API условного Todoist для заведения тасок

## Потенциальный data flow
**Не делегируется LLM**
- Конвертация аудио/видео в текст (выполняется отдельной STT-моделью Whisper)
- Отправка POST-запросов в таск-трекер (выполняется Python-кодом после подтверждения)

**Делегируется LLM/Агенту**
- Извлечение сущностей (кто, что обещал, сроки) и построение графа связей из сырого текста
- Формирование итоговой JSON-структуры карточки задачи для API
- Ответы на вопросы пользователя по базе прошлых встреч (RAG)
