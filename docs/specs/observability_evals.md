# Spec: Observability / Evals

## Метрики

Собираются локально через Loguru + кастомный счетчик токенов. Во внешние системы не отправляются.

### Технические метрики (per request)

| Метрика | Описание |
| :--- | :--- |
| `latency_total_ms` | Полное время обработки от загрузки файла до показа результата (без whisperx) |
| `latency_llm_ms` | Время каждого LLM-вызова по узлам |
| `latency_whisperx_ms` | Время транскрибации (отдельно, не в SLO) |
| `tokens_input` | Входящие токены за сессию |
| `tokens_output` | Исходящие токены за сессию |
| `cost_rub` | Расчетная стоимость сессии в рублях |
| `retry_count` | Количество retry по узлам |
| `refinement_count` | Количество итераций refine |

### Продуктовые метрики (per session)

| Метрика | Описание | Цель |
| :--- | :--- | :--- |
| `tasks_approved_without_edit` | Доля задач, принятых без изменений | > 80% |
| `meeting_type_classified` | Тип встречи, определенный классификатором | - |
| `session_outcome` | `confirmed` / `rejected` / `error` | - |

## Логи

Формат: JSON-lines, Loguru.

```python
# Пример лог-записи
{
  "timestamp": "2026-03-31T12:00:00Z",
  "session_id": "abc-123",
  "node": "work_extractor",
  "level": "INFO",
  "event": "extraction_complete",
  "tokens_input": 18420,
  "tokens_output": 312,
  "latency_ms": 4200,
  "retry_count": 0
}
```

Что **не логируется**: текст транскрипта, текст промптов, текст ответов LLM

Что логируется: метрики токенов, latency, коды ошибок, retry, session_id, meeting_type

Файл логов: `logs/meetaction.jsonl`, ротация по размеру 10 MB

## Evals

### Eval 1 - Качество классификации типа встречи

| Параметр | Значение |
| :--- | :--- |
| **Метод** | LLM-as-judge: отдельный вызов оценивает корректность классификации |
| **Датасет** | 10 синтетических транскриптов (5 work, 5 consultation) |
| **Метрика** | Accuracy >= 0.90 |

### Eval 2 - Качество извлечения задач (work_meeting)

| Параметр | Значение |
| :--- | :--- |
| **Метод** | Сравнение извлеченных задач с эталоном (hand-labeled) |
| **Датасет** | 5 синтетических транскриптов рабочих созвонов |
| **Метрики** | Precision / Recall по полям `assignee`, `action`, `deadline` |
| **Цель** | Precision >= 0.80, Recall >= 0.75 |

### Eval 3 - Продуктовая метрика (пользовательский approve)

| Параметр | Значение |
| :--- | :--- |
| **Метод** | Ручная оценка на 10 реальных / синтетических встречах |
| **Метрика** | % задач принятых без редактирования |
| **Цель** | >= 80% |

### Eval 4 - Latency SLO

| Параметр | Значение |
| :--- | :--- |
| **Метод** | Замер p95 latency на 20 тестовых запусках |
| **Метрика** | p95 latency агента (без whisperx) |
| **Цель** | <= 30 сек |
