## Политика логов и работы с персональными данными (PII)
* Транскрипты встреч не сохраняются в постоянную базу после формирования задач (в рамках PoC)
* Перед отправкой транскрипта в API LLM используется локальный NER-фильтр/регулярки для маскирования чувствительных данных (номера телефонов, паспортов, финансовые показатели компании), заменяя их на токены `[PHONE]`, `[FINANCE]`
* Логи промптов, ответов LLM и latency хранятся локально, метрики токенов собираются для контроля бюджета, но сами тексты не уходят в сторонние системы аналитики

## Защита от prompt injection и подтверждение действий
* Системные промпты изолированы от пользовательских данных (транскриптов). Транскрипт подается исключительно как анализируемый текст в тегах `<transcript>...</transcript>`
* **human-in-the-loop** - агент функционирует в режиме "read-only" до тех пор, пока не получит явное подтверждение от пользователя. Вызов инструмента `create_task_tool` физически заблокирован на уровне кода без флага подтверждения в стейте.
